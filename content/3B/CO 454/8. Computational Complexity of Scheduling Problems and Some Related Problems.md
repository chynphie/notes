> [!definition] P (Polynomial-time)
> P:=$\{ \pi \text{ is a decision problem}:\:\exists\text{ a polynomial time algo. for solving}\:\pi \}$ 
> 
> The class P contains all decision problems for which there exists a Turing machine algorithm that leads to the right yes-no answer in a number of steps bounded by a polynomial in the length of the encoding.

- I.e. There exists an algo. For solving the decision prob $\Pi$ s.t. For every input I for $\Pi$, Alg(I) returns the correct answer after O(size(I)$^{k}$) elementary arithmetic operations, where k is an absolute constant. 
- Note that for the alg. Alg to be a polynomial itme alg. It's not sufficient that the number of airthmetic operations performed by Alg(I) is bounded by a function $O([size(i)]^{k})$, but it is also necc. That the size of the numbers generated throughout the execution of the algo are also bounded by O([size(i)]$^{k}$). This detailed model of computation is called <u>bit complexity model on a turing machine</u>

> - LP, Shortest s-t path, Assignment problem, Optimal matching

- Polynomial-time proof-verification algorithm
	- size(proof(I)) = O($[size(I)]^{k}$)
		- (Because if the **proof** were _super large_ (e.g., exponential), then even reading or verifying it wouldn't be possible in polynomial time. So to keep the **verification process** efficient (i.e., polynomial time), we must also restrict the **length of the proof** to be polynomial in the size of the input.)
	- running time of Alg-verify $[I,\:proof(I)]$ = $O([size(I)]^{k})$, k is an absolute constant

> [!definition] NP (non-deterministic polynomial-time)
> NP (non-deterministic polynomial-time) := {
> - $\exists$ a poly. Time proof-verification algo, Alg-verify, for $\pi$ s.t. 
> 	- For every YES instance I of $\pi$, $\exists$ a concise proof(I) for which Alg-verify returns YES, and 
> 	- for every NO instance of I, there does not $\:\exists$ any second input for which Alg-verify returns YES
> 	}
> 
> 	The class NP contains all decision problems for which the correct answer, given a proper clue, can be verified by a Turing machine in a number of steps bounded by a polynomial in the length of the encoding.

> [!warning] Note
> $$
> P\subseteq NP$$ 
> Since $\forall \pi \in P$, the polynomial time algo for $\pi$ can be used as a proof-verification alg. Without the second input proof(I)

> [!definition] NP-hard 
> A problem P, either a decision problem or an optimization problem, is called NP-hard if the entire class of N P problems polynomially reduces to P.

- This $w$ is the "**second input**" mentioned in the note:
	- **First input** = the instance of the problem $I$
	- **Second input** = the proof $w$ (the proposed solution)

- Proof-verification algorithm is an algorithm that takes:
	- the original problem instance $I$
	- a proposed proof/solution $w$
- And checks whether $w$ is a **valid solution** to $I$, all in **polynomial time**.

> [!definition] NP-complete
> NP-complete := $\{ \bar{\pi}\in NP:\pi\:\alpha\:\bar{\pi}\:for\:every\:\pi \in NP \}$

### Satisfiability (SAT)
